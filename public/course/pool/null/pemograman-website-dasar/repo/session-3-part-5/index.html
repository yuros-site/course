Struktur Dasar HTML

null
13 min red
Januari 01, 0001

Struktur dasar HTML adalah kerangka utama yang membentuk sebuah halaman web. Setiap halaman yang dibuat menggunakan HTML memiliki pola susunan elemen tertentu yang harus diikuti agar dapat dibaca dan dipahami oleh browser. Susunan ini berfungsi seperti fondasi pada sebuah bangunan, di mana tanpa fondasi yang tepat, seluruh komponen di atasnya tidak dapat berdiri dengan baik.

Komponen utama dalam struktur dasar HTML terdiri dari deklarasi <!DOCTYPE html>, elemen <html>, <head>, dan <body>. Masing-masing bagian memiliki fungsi khusus yang saling melengkapi. Deklarasi doctype memberi tahu browser jenis dokumen yang digunakan, sementara tag <html> menjadi wadah bagi seluruh konten halaman. Dengan memahami fungsi tiap elemen, kita dapat menulis dokumen HTML yang rapi, valid, dan mudah dikembangkan.

Bagian <head> dalam struktur HTML digunakan untuk menyimpan metadata, seperti judul halaman, pengaturan karakter, dan tautan ke file pendukung seperti CSS atau JavaScript. Metadata ini tidak langsung terlihat oleh pengguna, tetapi sangat penting untuk pengaturan tampilan, optimasi mesin pencari (SEO), serta aksesibilitas. Sebaliknya, bagian <body> berisi semua konten yang terlihat oleh pengguna, seperti teks, gambar, tautan, dan elemen interaktif lainnya.

Dengan menguasai struktur dasar HTML, kita akan mampu menyusun halaman web sederhana yang sesuai standar. Pemahaman ini menjadi langkah awal sebelum melanjutkan ke tahap berikutnya, seperti menambahkan gaya visual menggunakan CSS atau menciptakan interaktivitas melalui JavaScript. Struktur yang benar tidak hanya memudahkan pengembangan, tetapi juga memastikan halaman dapat diakses dengan baik di berbagai perangkat dan browser.

1 Struktur dasar dokumen html

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Struktur Dasar HTML</title>
</head>
<body>
    <h1>Judul Halaman</h1>
    <p>Ini adalah contoh dokumen HTML sederhana.</p>
</body>
</html>

Struktur ini digunakan hampir oleh seluruh layanan yang menggunakan web browser, mulai dari instagram sampai dengan netflix. Mari kita bahas satu persatu untuk pemahaman yang lebih baik.

2. Doctype

Sebuah halaman HTML selalu diawali dengan deklarasi <!DOCTYPE html>. Deklarasi ini berfungsi untuk memberi tahu browser bahwa dokumen yang sedang diproses mengikuti standar HTML5. Dengan adanya deklarasi tersebut, browser dapat menginterpretasikan kode sesuai aturan yang berlaku secara universal. Hal ini sejalan dengan standar yang ditetapkan oleh World Wide Web Consortium (W3C, 2016), di mana deklarasi doctype menjadi bagian penting untuk memastikan keseragaman interpretasi dokumen web di berbagai perangkat dan aplikasi peramban.

Jika deklarasi doctype tidak dituliskan, browser berpotensi masuk ke dalam mode yang disebut quirks mode. Pada mode ini, browser akan mencoba menampilkan halaman dengan cara lama yang sering kali tidak sesuai standar modern. Menurut Leach dan Dusseault (2006), quirks mode menyebabkan terjadinya inkonsistensi dalam tampilan halaman karena setiap browser dapat menafsirkan kode secara berbeda. Kondisi ini dapat mengganggu pengalaman pengguna dan menyulitkan pengembang dalam memastikan kompatibilitas lintas browser.

Penerapan deklarasi doctype yang benar tidak hanya memengaruhi tampilan halaman, tetapi juga berhubungan dengan praktik web development yang lebih luas. Misalnya, ketika pengembang menggunakan HTML5 dengan deklarasi doctype, mereka dapat memanfaatkan fitur terbaru seperti elemen semantik, integrasi multimedia, serta dukungan untuk API modern. Hal ini menunjukkan bahwa doctype bukan sekadar baris kode pembuka, melainkan pintu masuk menuju penggunaan teknologi web yang lebih canggih dan berstandar.

Sebagai contoh implementasi, kode HTML berikut menunjukkan penggunaan deklarasi doctype dengan standar HTML5:

<!DOCTYPE html>
<html>
</html>

Kode sederhana ini merupakan titik awal yang wajib dipahami sebelum membangun struktur HTML lebih lengkap. Dengan membiasakan diri menuliskan deklarasi ini, kita tidak hanya mengikuti aturan teknis, tetapi juga menanamkan kebiasaan menulis kode sesuai standar internasional, sebagaimana disarankan dalam literatur pengembangan web modern (Duckett, 2011).

2.1 Dengan Deklarasi Doctype (Mode Standar)

Browser akan merender halaman sesuai aturan HTML5 dan CSS modern, sehingga tampilan lebih konsisten antar browser.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh Doctype</title>
    <style>
        p {
            width: 200px;
            border: 2px solid blue;
            padding: 20px;
        }
    </style>
</head>
<body>
    <p>Teks dengan doctype.</p>
</body>
</html>

Hasilnya kotak paragraf akan terlihat dengan ukuran 200px + padding + border, sesuai standar CSS box model.

2.2 Tanpa Deklarasi Doctype (Quirks Mode)

Jika baris <!DOCTYPE html> dihapus, beberapa browser lama akan masuk ke quirks mode. Dalam mode ini, interpretasi box model sering berbeda: padding dan border dihitung di dalam lebar elemen.

<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh Tanpa Doctype</title>
    <style>
        p {
            width: 200px;
            border: 2px solid red;
            padding: 20px;
        }
    </style>
</head>
<body>
    <p>Teks tanpa doctype.</p>
</body>
</html>

Hasilnya paragraf tampak lebih sempit, karena lebar total dihitung tetap 200px termasuk padding dan border.

Dari perbandingan tersebut, kita dapat melihat bahwa deklarasi doctype sangat berpengaruh pada konsistensi tampilan halaman. Dengan menuliskan <!DOCTYPE html>, browser modern akan menggunakan standards mode, sehingga hasil tampilan sesuai aturan W3C. Tanpa doctype, halaman dapat terlihat berbeda pada browser yang berbeda, terutama versi lama.

3. html

Elemen <html> adalah akar dari sebuah dokumen HTML dan berfungsi sebagai wadah yang menampung seluruh konten dalam halaman web. Semua elemen lain, baik metadata maupun konten yang ditampilkan pada pengguna, harus diletakkan di dalam tag ini. Dengan demikian, <html> dapat dianggap sebagai kerangka utama yang mengikat keseluruhan struktur dokumen. Menurut W3C (2016), elemen ini merupakan titik awal yang memberi konteks pada browser untuk memproses isi dokumen dengan benar.

Selain sebagai akar, elemen <html> juga dapat diberi atribut khusus, salah satunya adalah lang. Atribut ini berfungsi untuk mendefinisikan bahasa utama dari dokumen, misalnya lang="id" untuk bahasa Indonesia atau lang="en" untuk bahasa Inggris. Fungsi ini bukan hanya bersifat dekoratif, tetapi memiliki dampak nyata pada aksesibilitas dan pengoptimalan mesin pencari (SEO). Penelitian oleh Yesilada et al. (2009) menunjukkan bahwa penggunaan atribut bahasa secara konsisten membantu screen reader dalam membacakan teks sesuai fonetik yang benar.

Dari perspektif SEO, atribut lang membantu mesin pencari mengindeks halaman dengan lebih akurat berdasarkan bahasa target. Hal ini sangat penting ketika sebuah website ditujukan untuk audiens multibahasa. Misalnya, tanpa atribut bahasa, Google dapat salah menafsirkan konteks konten sehingga mengurangi visibilitas situs pada hasil pencarian. Menurut Cutts (2010), praktik SEO-friendly coding termasuk penggunaan atribut bahasa yang konsisten mampu meningkatkan pengalaman pengguna sekaligus performa pencarian.

Contoh implementasi sederhana dari elemen <html> dengan atribut bahasa adalah sebagai berikut:

<!DOCTYPE html>
<html lang="id">
</html>

Kode di atas mendeklarasikan bahwa halaman ditulis menggunakan bahasa Indonesia. Dengan penerapan ini, pengembang bukan hanya menulis kode yang sesuai standar, tetapi juga memastikan halaman mereka lebih mudah diakses, dipahami, dan dioptimalkan untuk berbagai keperluan, baik teknis maupun pengalaman pengguna (Duckett, 2011).

Untuk memberikan pemahaman yang lebih baik, mari kita lihat contoh kasus untuk memperjelas bagaimana attribute memberikan pengaruh yang signifikan pada laman web. Sebagai contoh kita akan mengambil penggunaan atribut lang="en" dan lang="id" pada elemen <html> dalam 2 kasus dibawah ini.

3.1 Screen Reader

Bayangkan sebuah halaman artikel ditulis dalam bahasa Indonesia tetapi elemen <html> tidak diberi atribut lang. Saat pengguna tunanetra membuka halaman tersebut menggunakan screen reader dengan pengaturan bahasa default bahasa Inggris, maka teks akan dibacakan dengan fonetik bahasa Inggris. Misalnya kata “belajar HTML” bisa terdengar salah pelafalan, sehingga makna sulit dipahami. Jika halaman diberi atribut lang="id", screen reader akan otomatis membaca dengan fonetik bahasa Indonesia, sehingga aksesibilitas meningkat (Yesilada et al., 2009).

Contoh kode yang benar untuk bahasa Indonesia:

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Artikel Belajar HTML</title>
</head>
<body>
    <p>Selamat datang di artikel belajar HTML dasar.</p>
</body>
</html>

3.2. Search Engine Optimization (SEO)

Misalnya sebuah situs berita internasional memiliki dua versi halaman: satu dalam bahasa Inggris dan satu dalam bahasa Indonesia. Jika halaman bahasa Indonesia ditulis tanpa lang="id", Google bisa salah mendeteksi bahasa konten sebagai bahasa Inggris karena banyak istilah teknis berbahasa Inggris di dalamnya. Akibatnya, halaman tersebut mungkin tidak muncul optimal di hasil pencarian untuk pengguna yang mencari konten berbahasa Indonesia. Dengan menambahkan lang="id", mesin pencari akan lebih mudah memahami konteks bahasa dan menampilkan halaman tersebut kepada audiens yang tepat (Cutts, 2010).

Contoh kode yang benar untuk bahasa Inggris:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Learning HTML Basics</title>
</head>
<body>
    <p>Welcome to the basic HTML tutorial.</p>
</body>
</html>

3.3. Apa yang kita simpulkan ?

Perbedaan sederhana dalam penulisan atribut lang dapat memengaruhi baik aksesibilitas maupun SEO. Pada konteks aksesibilitas, atribut ini memastikan teks dibacakan dengan fonetik yang benar, mendukung inclusive design dalam pengembangan web. Sementara pada konteks SEO, atribut ini membantu mesin pencari mengindeks halaman sesuai bahasa target, meningkatkan peluang situs muncul pada audiens yang tepat. Oleh karena itu, penggunaan atribut lang sebaiknya dianggap sebagai praktik wajib dalam menulis dokumen HTML modern (Duckett, 2011).

Aspek Dengan lang attribute Tanpa lang attribute
Screen Reader Membaca teks dengan fonetik sesuai bahasa (contoh: belajar → [bə.la.jar]) sehingga mudah dipahami pengguna tunanetra. Membaca teks dengan fonetik default (misalnya bahasa Inggris), menyebabkan kata terdengar salah dan sulit dimengerti.
SEO (Mesin Pencari) Mesin pencari mengenali bahasa dokumen dengan tepat, meningkatkan visibilitas pada audiens target sesuai bahasa. Mesin pencari bisa salah mendeteksi bahasa, mengurangi relevansi di hasil pencarian multibahasa.
User Experience Pengguna internasional dapat diarahkan ke versi bahasa yang sesuai, meningkatkan kenyamanan akses. Pengguna bisa diarahkan ke halaman yang salah bahasanya, membingungkan dan menurunkan pengalaman pengguna.
Standarisasi W3C Sesuai rekomendasi W3C (2016) untuk praktik aksesibilitas dan internationalization. Tidak sesuai standar, berpotensi menimbulkan masalah aksesibilitas dan kompatibilitas di masa depan.

4. head

Bagian <head> pada dokumen HTML memiliki peran penting sebagai tempat penyimpanan informasi metadata. Metadata dapat diartikan sebagai data tentang data, yaitu informasi tambahan yang menjelaskan konten halaman web tanpa ditampilkan secara langsung kepada pengguna. Meskipun tidak terlihat, metadata sangat krusial untuk memastikan halaman dapat diproses dengan baik oleh browser, mesin pencari, maupun perangkat lunak lain yang membaca dokumen web. Menurut W3C (2016), penggunaan metadata yang tepat dapat meningkatkan konsistensi tampilan, kompatibilitas lintas perangkat, serta aksesibilitas halaman.

Salah satu elemen metadata yang wajib ada adalah <meta charset="UTF-8">. Elemen ini berfungsi untuk menentukan standar pengkodean karakter, sehingga teks dapat ditampilkan dengan benar dalam berbagai bahasa, termasuk bahasa dengan aksara non-Latin seperti Arab, Jepang, atau Indonesia. Tanpa pengaturan ini, sering kali karakter khusus muncul dalam bentuk simbol yang tidak terbaca. Kajian oleh Yague et al. (2014) menekankan pentingnya character encoding dalam menjaga keterbacaan konten pada lingkungan web yang multibahasa.

Selain itu, elemen <title> berperan memberikan judul halaman yang akan muncul di tab browser maupun hasil pencarian mesin pencari. Judul yang jelas dan relevan tidak hanya membantu pengguna mengenali isi halaman, tetapi juga meningkatkan kinerja SEO (Search Engine Optimization). Cutts (2010) menyebutkan bahwa penulisan judul halaman yang sesuai dengan isi konten merupakan salah satu praktik terbaik dalam meningkatkan visibilitas di mesin pencari. Dengan demikian, <title> dapat dianggap sebagai elemen kecil namun berdampak besar dalam pengalaman pengguna.

Elemen lain dalam <head> adalah <link> dan <script>. Elemen <link> biasanya digunakan untuk menghubungkan halaman dengan file eksternal, seperti stylesheet CSS, yang bertugas mengatur tampilan visual situs. Sementara itu, <script> digunakan untuk menyisipkan atau menghubungkan file JavaScript yang menambahkan interaktivitas pada halaman web. Kombinasi antara <link> dan <script> memungkinkan pengembang memisahkan struktur, gaya, dan fungsi halaman sesuai prinsip separation of concerns. Hal ini sejalan dengan panduan pengembangan web modern yang menekankan pentingnya modularitas dan keterbacaan kode (Duckett, 2011).

Isi umum dari <head> meliputi:

  • <meta charset="UTF-8"> → memastikan karakter dapat terbaca dengan benar.
  • <title> → judul halaman yang tampil pada tab browser.
  • <link> → menyambungkan file eksternal (misalnya CSS).
  • <script> → menyisipkan JavaScript.

Berikut contoh implementasi lengkap bagian <head>** dengan elemen-elemen yang sudah dijelaskan:

<!DOCTYPE html>
<html lang="id">
<head>
    <!-- Menentukan standar karakter -->
    <meta charset="UTF-8">

    <!-- Judul halaman (terlihat di tab browser & hasil pencarian Google) -->
    <title>Struktur Dasar HTML</title>

    <!-- Menghubungkan file CSS eksternal -->
    <link rel="stylesheet" href="style.css">


    <!-- Menyisipkan JavaScript eksternal -->
    <script src="script.js"></script>
</head>
<body>
    <h1>Halo, ini contoh halaman dengan elemen head lengkap!</h1>
</body>
</html>

Dari contoh kode diatas untuk memberikan pemahaman yang lebih baik bagi kita dari masing-masing elemen.

<meta> berfungsi untuk menentukan character encoding dokumen sehingga karakter dari berbagai bahasa (huruf Latin, aksara non-Latin, simbol) dapat ditampilkan dengan benar tanpa muncul karakter rusak. Untuk praktik terbaik, kita lebih baik menempatkan element <meta charset="UTF-8"> pada posisi pertama di dalam head <head> agar browser langsung mengetahui jenis encoding sebelum mulai melakukan pemorosesn teks dan resource lain. Jika tidak, sejumlah byte awal bisa salah diinterpretasikan oleh browser sehingga menampilkan informasi yang rusak bagi pengguna. Penggunaan UTF-8 direkomendasikan karena cakupannya luas dan kompatibel dengan hampir semua bahasa modern.

<title> berfungsi untuk menetapkan judul halaman yang muncul di tab browser, riwayat peramban, dan sering dipakai mesin pencari sebagai teks utama di hasil pencarian. Judul yang informatif, singkat, dan unik per halaman meningkatkan usability dan SEO — membantu pengguna menemukan dan membedakan halaman saat membuka banyak tab atau melihat hasil pencarian. Selain itu, beberapa teknologi pembaca layar menggunakan judul halaman untuk memberi konteks awal kepada pengguna.

<link> berfungsi untuk menghubungkan CSS eksternal yang mengatur tampilan visual halaman. Menaruh link stylesheet di <head> adalah praktik umum karena browser men-download CSS sebelum merender halaman sehingga meminimalkan flash of unstyled content (FOUC); namun CSS eksternal bersifat render-blocking sehingga perlu dipertimbangkan optimasi (mis. minify, combine, prefetch, atau media queries) untuk performa. Untuk resource dari CDN juga bisa menambahkan atribut keamanan seperti integrity dan crossorigin.

<script> berfungsi untuk menyisipkan JavaScript melalui script eksternal memungkinkan menambah interaktivitas dan logika aplikasi tanpa mencampur kode ke struktur HTML (prinsip separation of concerns). Perlu diketahui bahwa <script> tanpa atribut tambahan bersifat blocking: browser akan menghentikan parsing DOM sampai file script diunduh dan dieksekusi, sehingga menaruh script di akhir <body> sering direkomendasikan untuk menghindari penghambatan rendering.

5. body

Elemen <body> dalam HTML merupakan bagian utama yang menampung seluruh konten yang terlihat dan dapat diakses langsung oleh pengguna melalui browser. Semua teks, gambar, multimedia, maupun elemen interaktif diletakkan di dalam bagian ini sehingga ia berfungsi sebagai representasi visual halaman web. W3C (2016) menegaskan bahwa struktur dalam <body> harus ditata secara semantik agar mesin pencari, assistive technology, dan browser dapat memahami konteks setiap elemen dengan baik. Dengan demikian, <body> dapat dipandang sebagai inti komunikasi antara pengembang dengan audiensnya. Elemen-elemen umum dalam body antara lain:

  • Heading <h1> sampai <h6> yang biasanya digunakan sebagai struktur judul dan subjudul.
  • Paragraf <p> digunakan untunnk menulis paragraf.
  • Gambar <img> digunakan untuk menyisipkan gambar.
  • Tautan <a> digunakan untuk membuat navigasi antar halaman.

Salah satu elemen penting dalam <body> adalah heading <h1> sampai <h6>. Heading digunakan untuk memberi struktur hierarkis pada dokumen, mulai dari judul utama hingga sub-bagian terkecil. Menurut penelitian Baeza-Yates dan Ribeiro-Neto (2011), penggunaan heading yang konsisten tidak hanya meningkatkan keterbacaan manusia, tetapi juga membantu mesin pencari mengidentifikasi prioritas informasi dalam sebuah halaman. Oleh karena itu, kita perlu memahami bahwa heading bukan hanya masalah tampilan, melainkan juga instrumen penting dalam penyusunan dokumen web yang ramah SEO.

Elemen body yang terstruktur dengan baik akan meningkatkan aksesibilitas web, terutama bagi pengguna dengan keterbatasan. W3C (2016)

Selain heading, paragraf <p> digunakan untuk menyajikan teks isi, sementara gambar <img> menambah dimensi visual yang memperkaya pengalaman pengguna. Setiap elemen gambar wajib dilengkapi atribut alt sebagai alternatif teks. Hal ini bukan hanya membantu pengguna tunanetra dengan screen reader, tetapi juga menjadi pedoman aksesibilitas global yang ditekankan dalam Web Content Accessibility Guidelines (WCAG) versi 2.0 (W3C, 2016). Dengan menambahkan deskripsi alternatif, pengembang tidak hanya memenuhi standar teknis, tetapi juga mempraktikkan prinsip inklusivitas dalam pengembangan website.

Selain teks dan gambar, tautan <a> merupakan elemen fundamental untuk navigasi antar halaman maupun menuju sumber daya eksternal. Tautan menjadikan sebuah situs benar-benar “terhubung” dalam konteks World Wide Web. Namun, tautan harus disajikan dengan teks deskriptif, bukan sekadar kata seperti “klik di sini”, agar pengguna memahami tujuan dari tautan tersebut. Nielsen (2000) menekankan bahwa tautan yang jelas dan kontekstual akan meningkatkan usability serta membantu pengguna dalam menjelajahi situs dengan lebih efisien. Dengan demikian, pemahaman dan penerapan elemen-elemen di dalam <body> bukan sekadar teknis, melainkan praktik terbaik dalam membangun pengalaman web yang efektif dan inklusif.

Contoh implementasi:

<body>
    <h1>Selamat Datang di Website Saya</h1>
    <p>Ini adalah paragraf pertama yang menjelaskan tentang struktur dasar HTML.</p>
    <img src="gambar.jpg" alt="Contoh Gambar">
    <a href="https://www.example.com">Kunjungi Website Lain</a>
</body>

6. Kesimpulan

Dengan menguasai struktur dasar HTML, kita akan mampu membangun halaman web yang valid, dapat diakses, dan sesuai standar internasional. Seperti yang ditekankan oleh World Wide Web Consortium (W3C), mengikuti standar web adalah langkah awal menuju pengembangan aplikasi web yang inklusif dan berkelanjutan.
